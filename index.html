RL‑Guided Compiler Autotuning with Surrogate Forecasting

Teaching a policy to tune compiler schedules faster than the search itself can.

Bare‑metal ML acceleration lives or dies by the quality of its compiled schedules — tile shapes, loop orders, allocation choices. The standard answer is exhaustive or random search, which scales badly as workloads diversify. This project asked a sharper question: can a learned policy, guided by cheap surrogates of the expensive objective, find good schedules in a fraction of the search budget?

The framework couples a PPO agent with a cycle‑accurate hardware simulator in a tight loop, and layers a sequence‑based forecasting model on top of the partial tuning trajectory to predict terminal reward and cycle counts. Those forecasts drive two things: early stopping of unpromising rollouts, and candidate prioritization for the next step of search. The result is a co‑designed pipeline where the compiler, the hardware model, and the learning loop are all aware of one another.

  • Compiler–hardware co‑design. Policy actions operate over the real schedule space the simulator exposes, not an abstracted surrogate.
  • Workload‑family generalization. Evaluated against held‑out shapes and constraint regimes via replay traces and ablation.
  • Surrogate forecasting. Trajectory‑conditioned predictors cut wall‑clock tuning time while keeping final schedule quality within a tight margin of exhaustive baselines.

BioLens — a biomedical knowledge graph for research exploration

Diseases, drugs, genes, and trials — integrated, queryable, and visual.

Biomedical researchers routinely reach for four or five different public data sources just to check a single hypothesis — PubMed for literature, OpenFDA for adverse events, Open Targets for target–disease evidence, ClinicalTrials.gov for trial records. BioLens stitches those into a single exploratory surface: a force‑directed knowledge graph you can pan through, filter, and expand, backed by an LLM agent that speaks the same schema.

Under the hood, an async FastAPI backend fans out concurrent queries to the four source APIs, normalizes entities, and stores the results in a graph store. The frontend is a Next.js/TypeScript app that renders the network in D3 with zoom, pan, and click‑to‑expand on any entity, plus Vega‑Lite charts for statistical summaries. The interesting layer is the MCP server that exposes six research tools to an LLM‑based coding agent — natural‑language queries turn into schema‑aware graph updates in the UI.

  • Four sources, one graph. Async concurrent ingestion across PubMed, ClinicalTrials.gov, OpenFDA, and Open Targets.
  • MCP‑exposed research tools. Six tools let a coding agent operate on the graph natively — chat becomes a first‑class query surface.
  • Shipped cleanly. Dockerized end‑to‑end, GitHub Actions CI/CD, open source from day one.
